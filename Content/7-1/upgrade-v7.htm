Upgrading to V7.0

In V7.0, you can now manage your s and dashboards anytime, anywhere from a centralized location inside the .

For an overview of V7’s features and functionality, see Getting Started with V7.

During the initial phases of the migration to a fully-online , not all of the functionality included in the desktop version of the will be available. All features will be available on the web early Q1 2018.

In the meantime, to overcome any gaps in your workflows, V7.0 supports two versions of the , the desktop version of the and the web-based version of the .

This page describes the implications of working with two versions of the and how both versions of the can work together to support backwards compatible features.

Working Side-by-Side FAQs

  • Can I open my existing s in the Web?
    Yes, your existing s will be displayed in the desktop version of the as well as the web-based version of the .
  • Do changes in the Web affect my on the Desktop and vice versa?
    Any changes you make from either version of the will be reflected in the other version after you build the .
  • Can I save a copy of my in the Web?
    In the new web-based version of the , your work is saved automatically. However, your changes are not reflected in your dashboard or the desktop version of the until the is built.
  • What functionality in the desktop version of the is not supported in the Web?
    Most features and functionality included in the desktop version of the are supported on the web, however, are few features have not yet been migrated.
    For a full list of features not currently supported and how to overcome these gaps, see V7.0 Backwards Compatibility below.
  • When will all desktop functionality be available in the Web?
    All functionality available in the desktop version of the will be available online by the end of the first quarter, 2018.
  • Is the desktop ECM going to be discontinued?
    No, the desktop version of the will continue to be installed with throughout 2018. If there are any plans to depreciate the desktop version after 2018, these will be communicated well in advanced.

V7.0 Backwards Compatibility

While most functionality is included in the web-based version of the , the following features will be available on the web from early Q1 2018:

  • Pre/Post Build Plugins
  • Import/Export ElastiCubes
  • Attach/Detach ElastiCubes
  • Change Connectivity
  • Add Data from Unsupported Data Sources

This section provides solutions to help you complete your tasks by working with two versions of the .

Pre/Post Build Plugins

When you create an in the web-based version of the , all pre-post plugins are disabled.

To enable pre/post build plugins to work with s created online:

  1. In Windows, open the Server Console.
  2. Open the relevant and select Edit. The is opened in the desktop version of the .
  3. Add your pre or post-plugins. For more information, see Pre- and Post-Build Plugins.
  4. Build the in the desktop version of the . Note: Any unbuilt changes in the on the web-based version of the will be discarded.

Import/Export s

Import or Exporting s can only be done through the desktop version of the in the Server Console.

To export:

  1. In Windows, open the Server Console.
  2. Select the relevant , stop it, and then select Export.

To import:

  1. In Windows, open the Server Console.
  2. Click the Import button and select the relevant file to import. Note: s with the same name will be overwritten and replaced. Any unbuilt changes in the in the web-based version of the will be discarded.
  3. Refresh the Data page in the web-based version of the to access the imported .

Attach-Detach

Attaching or detaching s can only be done through the desktop version of the in the Server Console.

To detach an :

  1. In Windows, open the Server Console.
  2. Select the relevant and select Detach.
  3. Refresh the Data Page in your browser and continue to work if needed. Any dashboards using the detached will not function.

To attach an :

  1. Refresh the Server Console.
  2. In the Server Console, click the Attach button and the relevant file to attach. Note: s with the same name will be overwritten and replaced. Any unbuilt changes in the in the web-based version of the will be discarded.
  3. Refresh the Data Page in your browser to access the attached and continue to work if needed.

Change Connectivity

Change connectivity is not available for s built in the web-based version of the .

To change connectivity settings for an build in the web-based version of the :

  1. In Windows, open the Server Console.
  2. Select the relevant and select Edit. The is opened in the desktop version of the .
  3. In the select s > Change connectivity.
  4. Change the relevant connectivity settings. For more information see Changing Connectivity Settings for Data Sources.
  5. Perform a build from the desktop . Note: Any unbuilt changes on the in the web-based version of the will be discarded.
  6. Refresh the Data Page in your browser and continue to work as needed.

Add Data from an Unsupported Data Source

Currently, the web-based version of the supports data from CSV files, Excel spreadsheets, Oracle, SQL Server, MySQL, Google Sheets, RedShift and Postgres SQL (support for additional connectors will be released monthly), however, the web-based version of the supports all existing data sources to view and edit. This includes the ability to:

  • Add/edit/delete new columns and custom SQL tables
  • Create/edit/delete relationships
  • Perform any table and field operations: rename, duplicate, delete, preview
  • Hide, import, or index fields
  • Set table level build settings

If you need to work with data from a source that is not currently supported, you can import the data into the desktop version of the and continue to work on the online after the data has been added.

To add data from unsupported data sources:

  1. In Windows, open the Server Console.
  2. Select the relevant and select Edit. The is opened in the desktop version of the .
  3. In the desktop version of the , add the relevant tables from any of the data sources supported in the desktop version of the .
  4. Perform a build from the desktop . Note: Any unbuilt changes in the on the web-based version of the will be discarded.
  5. Refresh the Data Page in your browser and continue to work as needed.
