Changing the ElastiCube Before Loading a Dashboard

This topic explains how to dynamically set an ElastiCube before loading a dashboard with the Dynamic add-on.

If you have multiple ElastiCubes with the same schema (perhaps split by customer), you may want to create a set of dashboards and pick which ElastiCube to query against according to which user logs in. Normally, you would have to create a dashboard for each ElastiCube to accomplish this, but this add-on allows you to create a single dashboard (or set of dashboards) and dynamically set the ElastiCube when the user loads the web page.

Note: All ElastiCubes used must have the same schema.

To change your s dynamically: 

  1. Download the attachment and unzip the contents into your C:\Program Files\Sisense\PrismWeb\plugins\ folder. If the plugins folder doesn't exist, just create it.
  2. Open up C:\Program Files\Sisense\PrismWeb\plugins\dynamicElasticube\settings.js and replace the default information.
    • server: This should be the URL of your Sisense server.
    • dashboards: This should be either a list of dashboards where this add-on will be applied, or enter ["all"] to apply this add-on to every dashboard.
    • elasticubes:  This should be a list of ElastiCubes that have an identical schema, such as ["Customer-A-Cube", "Customer-B-Cube", "Customer-C-Cube"].
    var dynamicElasticubeSettings = {
      //	Server info 
       server: "http://localhost:8081",
      //	Provide a list of dashboards to apply this plugin to
      dashboards: ["Dynamic Elasticubes"],	//	Apply to a specific list of dashboards
     //dashboards: ["all"]			//	Apply to all dashboards
     //	Provide a list of elasticubes that are available for switching
     //	Note that all elasticubes in this list must have the exact same schema
     elasticubes: ["Northwind2", "Northwind3"],
     //	Leave as is
     allElasticubes: []
};
  3. Create a set of dashboards built from one of the Elasticubes in the above list. If you specified a list of dashboards in settings.js, make sure the dashboards you create are in that list.
  4. Access the dashboards. When accessing your dashboards, append the new name to the end of the URL. The add-on will look for the name and switch the dashboard to use the new instead.
http://localhost:8081/app/main#/dashboards/5575cfac627dc10c2700001e?elasticube=Northwind3

This dashboard is built from an named Northwind3, which has data for North America.

The same dashboard is now querying against an named Northwind2, which has data for South America.
