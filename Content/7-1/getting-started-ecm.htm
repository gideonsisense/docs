Getting Started: New ElastiCube Manager

In Sisense V7.0, you can now manage your ElastiCubes and dashboards anytime, anywhere from a centralized location inside the Sisense Web Application.

During the initial phases of the move to a browser-based ElastiCube Manager there are a few functions only supported in the desktop version of the ElastiCube Manager. All features will be available on the web by early Q2 2018.

In the meantime, to fully support your workflows, Sisense V7.0 includes both the desktop version of the ElastiCube Manager and the browser-based version of the ElastiCube Manager.

This page describes the implications of working with two versions of the ElastiCube Manager and how both versions of the ElastiCube Manager can work together to support backwards compatible features.

Tips to Start

Below are recommendations to work with the new browser-based ElastiCube Manager.

  1. Familiarize yourself with the new ElastiCube Manager features and backwards compatibility.
  2. If available, install in a development environment to review and then move to production. Alternatively, upgrade your production environment.
  3. (Initially the browser based ElastiCube Manager is only shown to Administrators.)
  4. To understand new features first create a new ElastiCube in the browser-based ElastiCube Manager. A good example to follow is our product walk-through and video tutorials.
  5. Once comfortable with the features, duplicate an existing ElastiCube and open it in the browser based ElastiCube Manager. To open the cube in the browser simply refresh the Data Page.Make any changes in the cube such as creating or editing relationships, custom columns/tables, metadata and build these changes.Review the impact of these changes in the desktop. To do this open the server console and open the cube.
  6. Determine which cubes to use in the new browser environment. The browser-based Manager supports all existing data sources to view and edit. However, adding new tables is only supported from CSV files, Excel spreadsheets, Oracle, SQL Server, MySQL, RedShift, Google Sheets and Postgres SQL (additional connectors are released monthly).How frequently data needs to be added from currently unsupported data sources is the main factor to determine which cubes to base in the new browser environment. Learn more about compatibility below.

For additional advice, consult your Customer Success Manager and onboarding consultants.

Working Side-by-Side FAQs

  • Can I open my existing ElastiCubes in the Web?
    Yes, your existing ElastiCubes will be displayed in the desktop version of the ElastiCube Manager as well as the web-based version of the ElastiCube Manager.
  • What are the new user roles?
    Data Admin: Has full administrative rights to all ElastiCubes, but does not have rights to any other administrative features in the Admin page.
    Data Designer: Can create, share, and manage ElastiCubes, including ElastiCubes that have been shared with them.
    For more information about all Sisense user roles, see Sisense User Roles.
  • How are the new roles reflected in licensing?
    Defining a user as a Data Designer will count as 1 existing Designer license.
    Defining a user as a Data Admin will count as 1 existing Admin license.
  • Do changes in the Web affect my ElastiCube on the Desktop and vice versa?
    Any changes you make from either version of the ElastiCube Manager will be reflected in the other version only after you build the ElastiCube.
  • Can I save a copy of my ElastiCube in the Web?
    In the new web-based version of the ElastiCube Manager, your work is saved automatically. However, your changes are not reflected in your dashboard or the desktop version of the ElastiCube Manager until the ElastiCube is built.
  • How do I share an ElastiCube?
    Go to Admin> Data Management > Data Sources. Find the relevant ElastiCube, click on the menu to the far right and select Share.
  • What functionality in the desktop version of the ElastiCube Manager is not supported in the Web?
    Most features and functionality included in the desktop version of the ElastiCube Manager are supported on the web, however, are few features have not yet been migrated.
    For a full list of features not currently supported and how to overcome these gaps, see Sisense V7.0 Backwards Compatibility below.
  • When will all desktop functionality be available in the Web?
    All functionality available in the desktop version of the ElastiCube Manager will be available online by the end of the first quarter, 2018.
  • Is the desktop ECM going to be discontinued?
    No, the desktop version of the ElastiCube Manager will continue to be installed with Sisense throughout 2018. If there are any plans to depreciate the desktop version after 2018, these will be communicated well in advanced.

Sisense V7.0 Backwards Compatibility

While most functionality is included in the web-based version of the ElastiCube Manager, the following features will be available on the web from early Q1 2018:

  • Plugins
  • Import/Export ElastiCubes
  • Attach/Detach ElastiCubes
  • Change connectivity
  • Add Data from Unsupported Data Sources

This section provides solutions to help you complete your tasks by working with two versions of the ElastiCube Manager.

Plugins

When you create an ElastiCube in the web-based version of the ElastiCube Manager, all pre-post plugins are disabled.

To enable plugins to work with ElastiCubes created online:

  1. In Windows, open the Sisense Server Console.
  2. Open the relevant ElastiCube and select Edit. The ElastiCube is opened in the desktop version of the ElastiCube Manager.
  3. Add your pre or post-plugins. For more information, see Pre- and Post-Build Plugins.
  4. Build the ElastiCube in the desktop version of the ElastiCube Manager.
    Note: Any unbuilt changes in the ElastiCube on the web-based version of the ElastiCube Manager will be discarded.

Import/Export ElastiCubes

Import or Exporting ElastiCubes can only be done through the desktop version of the ElastiCube Manager in the ElastiCube Server Console.

To export:

  1. In Windows, open the Sisense Server Console.
  2. Select the relevant ElastiCube, stop it, and then select Export.

To import:

  1. In Windows, open the Sisense Server Console.
  2. Click the Import button and select the relevant file ElastiCube to import.
    Note: ElastiCubes with the same name will be overwritten and replaced. Any unbuilt changes in the ElastiCube in the web-based version of the ElastiCube Manager will be discarded.

  3. Refresh the Data page in the web-based version of the ElastiCube Manager to access the imported ElastiCube.

Attach-Detach ElastiCube

Attaching or detaching ElastiCubes can only be done through the desktop version of the ElastiCube Manager in the ElastiCube Server Console.

To detach an ElastiCube:

  1. In Windows, open the Sisense Server Console.
  2. Select the relevant ElastiCube and select Detach.
  3. Refresh the Data Page in your browser and continue to work if needed. Any dashboards using the detached ElastiCube will not function.

To attach an ElastiCube:

  1. Refresh the Sisense Server Console.
  2. In the Sisense Server Console, click the Attach button and the relevant file ElastiCube to attach.
    Note: ElastiCubes with the same name will be overwritten and replaced. Any unbuilt changes in the ElastiCube in the web-based version of the ElastiCube Manager will be discarded.
  3. Refresh the Data Page in your browser to access the attached ElastiCube and continue to work if needed.

Change Connectivity

Change connectivity is not available for ElastiCubes built in the web-based version of the ElastiCube Manager.

To change connectivity settings for an ElastiCube build in the web-based version of the ElastiCube Manager:

  1. In Windows, open the Sisense Server Console.
  2. Select the relevant ElastiCube and select Edit. The ElastiCube is opened in the desktop version of the ElastiCube Manager.
  3. In the ElastiCube Manager select ElastiCubes > Change connectivity.
  4. Change the relevant connectivity settings. For more information see Changing Connectivity Settings for Data Sources.
  5. Perform a build from the desktop ElastiCube Manager.
    Note: Any unbuilt changes on the ElastiCube in the web-based version of the ElastiCube Manager will be discarded.
  6. Refresh the Data Page in your browser and continue to work as needed.

Add Data from an Unsupported Data Source

Currently, the web-based version of the ElastiCube Manager supports data from CSV files, Excel spreadsheets, Oracle, SQL Server, MySQL, Google Sheets, RedShift and Postgres SQL (support for additional connectors will be released monthly), however, the web-based version of the ElastiCube Manager supports all existing data sources to view and edit. This includes the ability to:

  • Add/edit/delete new columns and custom SQL tables
  • Create/edit/delete relationships
  • Perform any table and field operations: rename, duplicate, delete, preview
  • Hide, import, or index fields
  • Set table level build settings

If you need to work with data from a source that is not currently supported, you can import the data into the desktop version of the ElastiCube Manager and continue to work on the ElastiCube online after the data has been added.

To add data from unsupported data sources:

  1. In Windows, open the Sisense Server Console.
  2. Select the relevant ElastiCube and select Edit. The ElastiCube is opened in the desktop version of the ElastiCube Manager.
  3. In the desktop version of the ElastiCube Manager, add the relevant tables from any of the data sources supported in the desktop version of the ElastiCube Manager.
  4. Perform a build from the desktop ElastiCube Manager.
    Note: Any unbuilt changes in the ElastiCube on the web-based version of the ElastiCube Manager will be discarded.
  5. Refresh the Data Page in your browser and continue to work as needed.
