Deleting s

When you no longer want to use an , you can delete the from the Data page.

Deleting an removes it from the server and the ’s folder saved locally on your hard drive. After an has been deleted, it cannot be restored.

If your has any dependencies, or entities such as dashboards, that are currently connected to the , some of these entities will also be deleted and some you will be prompted to remap to another data source. Entities that are deleted when you delete an include:

  • Data security rules
  • Build alerts
  • Drill hierarchies

Entities that are not deleted, but should be remapped to another data source include:

  • Dashboards
  • Widgets (from other dashboards)
  • Pulse Alerts

Entities that you can remap to another data source are clickable links that when clicked, display a list of all the entities that need to be remapped.

You can choose to remap the entities to another data source manually or select the new data source from the list on the bottom of the Delete window. If you do not remap the entities, the dependencies will not work until you apply a new data source.

Note: Any temporary folders that were saved locally during failed builds are not deleted when the ElastiCube is deleted and must be removed manually.

  1. In the , click Data.
  2. From the menu of the relevant , select Delete.
  3. Click Yes to confirm you want to delete the .
    OR
    If you has any dependencies, remap those entities to a new or select None to remap them later.
