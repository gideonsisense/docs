Routing Queries in Sets

When a user queries an that is part of an Set, routes the query to a single with the latest build. By default, Sets operate in active-passive mode. In active-passive mode, one , by default the with the latest build, handles incoming queries. This is active while the remaining s are on standby in passive mode.

also supports active-active mode where queries are spread across available s within an Set. In active-active mode, each that is not currently building can handle queries.

You can modify how queries are routed to s within Sets by selecting one of the following routing strategies:

Single: (Default). s operate in active-passive mode where the with the freshest data receives all the queries while the remaining s are on standby.

Multiple s – Fresh Data: s operate in active-active mode where queries are spread across all s with the latest data.

Multiple s – Best Spread: s operate in active-active mode where queries are spread across the largest group of s with identical data.

To set a query routing strategy:

  1. Click Admin and select the Data Sources tab on the left.
  2. For the relevant Set, select Edit.
  3. From the Routing Strategy list, select the relevant strategy.
  4. Click Save.
