Hosting the Application Database

If you wish to host the application database on another server, you need to create an AppUser that Sisense can use to communicate with your application database. The AppUser you create must have two application database roles assigned to it, readWriteAnyDatabase and dbAdminAnyDatabase. These roles provide the AppUser read/write privileges to the application database.

In addition, the password for the AppUser should be encrypted using the same encryption mechanism uses. You can encrypt a plaintext password using this encryption mechanism in the REST API through the endpoint:

GET /app_database/encrypt_database_password

This encrypted password should be included in all the relevant configuration files as described in Connecting to the Replica Set.
