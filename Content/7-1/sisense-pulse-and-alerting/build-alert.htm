Creating Build Alerts

Administrators can set build alerts to notify themselves or other users when a build fails or succeeds. Build alerts are created from the and are sent following an build.

Build alerts allow you to monitor the status of builds.

Administrators can create build alerts from the Admin page of the . After a build alert is created, it is added to Pulse from where you can manage it.

To create a build alert:

  1. In the , select Admin > Data Sources > Add Build Alert.
    OR
    From the menu, Add Build Alert.

    The Add Build Alert window is displayed.
  2. In the Name field, enter a name for the alert.
  3. From the Server list, select the server where the is hosted.
  4. From the list, select the (s) that triggers an alert following a build.
  5. Select when to notify you following a build. There are three conditions that when met, can trigger an alert:
    Build failed: The alert is sent after the build fails.
    Build succeeded: The alert is sent after the is built successfully.
    Build succeeded after failure: The alert is sent following a successful build after the previously failed to build.
  6. Click Save. The Alert is added to your Pulse page.
    For additional advanced configuration options, see Advanced Settings.
