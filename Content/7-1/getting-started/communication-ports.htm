Communication Ports

utilizes multiple communication ports. Some of the ports are used for communication between modules, and others are used for communication with remote locations.

When is installed as part of a Single machine deployment, ports used for communication between modules are considered internal ports, and do not need to be open in the firewall. When is installed in a Clustered deployment, where modules installed on different computers communicate with each other, ports used for communication between modules may need to be opened in the firewall. This depends on the specific deployment implemented in your environment.

Note: When trying to add an server to a web server, Sisense pings the web server. If it cannot successfully ping the server, the error message "The Elasticube server is not reachable" is returned. Make sure that ICMP communication is enabled on the web server.

The tables below lists the ports on which listens for communication.
