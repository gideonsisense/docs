Sisense Remote Support Analysis Module

Sisense collects data when you or your users install or use Sisense products or services. Sisense uses data collected for internal and support-related purposes such as improving our products and services, improving customer engagement, conducting research, and resolving technical issues.

This page describes how to deactivate the service and how to obfuscate data sent to Sisense.

Deactivating the Sisense Remote Support Module

You can deactivate the Sisense Remote Support Module manually by editing a configuration file installed on your computer when you installed Sisense.

Keep in mind, by deactivating the Sisense Remote Support Module, Sisense Technical Support may not be able to troubleshoot any issues efficiently without the logs generated by the module.

To deactivate the Sisense Remote Support Module:

For Sisense V6.7 and later:

  1. In a text editor, open the conf file located at: …\ProgramData\Sisense\Infra\Data\Shipper\Config
  2. Comment out the following lines by adding # at the beginning of the line as shown below:
    #if “com” not in [sisenseuid] {
    # tcp {
    # host => “listener.logz.io”
    # port => “5052”
    # codec => “json_lines”
    # ssl_enable => true
    # ssl_verify => true
    # ssl_cert => “C:/ProgramData/Sisense/Infra/Data/Shipper/Config/keys/logstash.crt”
    # ssl_key => “C:/ProgramData/Sisense/Infra/Data/Shipper/Config/keys/logstash.key”
    # ssl_cacert => “C:/ProgramData/Sisense/Infra/Data/Shipper/Config/keys/TrustExternalCARoot.crt”
    # }
    #}
  3. Save the Configuration file.
  4. Open the Sisense.Oxygen.exe.config file located at …Program Files\Sisense\Infra\Oxygen\.
  5. Change the following value to false:
    <add key=”enableTracking” value=”false“/>
  6. Save the Configuration file.
  7. Open Microsoft Services and restart the Sisense.Oxygen service and the Sisense.Shipper Service.

For Sisense V6.6.1 and earlier:

  1. In a text editor, open the CustomApp.config file located at: …\ProgramData\Sisense\Infra\AgentConfig.
  2. Add the following line to the file between the <appSettings> tags:
    <add key=”run_shipper_outer” value=”false” />
  3. Save the Configuration file.
  4. Open the Sisense.Oxygen.exe.config file located at:
    …Program Files\Sisense\Infra\Oxygen\
  5. Change the following value to false:
    <add key=”enableTracking” value=”false“/>
  6. Save the Configuration file.
  7. Open Microsoft Services and restart the Sisense.Oxygen service and Sisense Agent Monitoring Service.

Scrambling Data Collected by the Sisense

If you do not want to deactivate monitoring, but rather protect sensitive information, you can obfuscate the information Sisense records. When you obfuscate your data, Sisense records random letters and numbers that are unreadable in place of the username. By default, the usernames of your users in Sisense are obfuscated.

Note: You can obfuscate additional fields such as email or ElastiCube name by adding the values email, ElastiCube delimited by a comma. For a complete list of all the data you can obfuscate, contact Sisense Support.

To scramble data recorded by the Sisense Remote Support Module:

For Sisense V6.7 and later:

  1. In a text editor, open the shipper.conf file located at …\ProgramData\Sisense\Infra\Data\Shipper\Config\shipper.conf
  2. Add the following line to the file between the <appSettings> tags:
    <add key=”fields_to_obfuscate” value=”userName” />
  3. Save the Configuration file.
  4. Open Microsoft Services and restart the Sisense.Shipper Service.

For Sisense V6.6.1 and earlier:

  1. In a text editor, open the CustomApp.config file located at …\ProgramData\Sisense\Infra\AgentConfig.
  2. Add the following line to the file between the <appSettings> tags:
    <add key=”fields_to_obfuscate” value=”userName” />
  3. Save the Configuration file.
  4. Open Microsoft Services and restart the Sisense Agent Monitoring Service.
